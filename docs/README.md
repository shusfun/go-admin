# 需求文档目录

`docs/` 现在用于存放业务需求相关材料，其中正式需求产出统一放在 `docs/requirements/`，原始记录可保留在 `docs/` 根目录。

当前约定：

- 正式需求、需求拆解、验收标准统一放在 `docs/requirements/`
- `docs/` 根目录用于存放原始记录、临时会议纪要和待整理材料
- 若后续需求需要按模块拆分，可在 `docs/requirements/` 下新增对应子目录
- `docs/admin-swagger/` 为 Swagger 生成产物目录，不作为需求文档入口
- 当前 [xuqiu.md](./xuqiu.md) 视为原始需求记录，暂不迁入正式需求目录

如果需要查看脚手架说明、开发工作流和专题设计，请转到 [../scaffold-docs/README.md](../scaffold-docs/README.md)。
